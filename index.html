Deadline types

Two kinds: weeks/days before event (deadline is calculated from the activity date) and predetermined date (fixed calendar date, e.g. hand in budgets on 1 August — independent of the activity date). Use {x} in a name for a fill-in placeholder (e.g. "Plan meeting with {x}" → x = ...), filled in per event type or per activity.

Post types (default time before event, in days)

Standard posts for the communication planner (name, channel, default days before the event — negative = after — and default posting time). Assign them per event type below, where the offset and time can be overridden.

Event types
Roles (praesidium functions)

Functions shown in step 1 of the deadline planner and used to filter the year-plan import (role column). The optional head responsible is the person currently leading that function; it is written to a fixed cell (B1) in the deadline Excel export. Renaming a role here does not update the role names in existing import files.

Communication channels

Channels suggested when building post plans (e.g. Instagram post, Mailing). Renaming a channel here does not change posts that already use the old name.

Academic year

Used by the communication exports to group everything per semester week.

🔗 Link page — generate events.json

Builds the events.json for the public link page (separate linktree repo). Import the same year-plan Excel as the planners (only activityName and activityDate are used), tick the events you want to promote, then download and upload the file to the linktree repository on GitHub. Ticket links are added later by editing the file directly on GitHub.

🚀 Publish directly to GitHub
